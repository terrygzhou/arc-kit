Technical 22 May 2026

How ArcKit tidies Wardley Map labels: a deterministic placement engine, stepped through

ArcKit renders every Wardley Map as a Mermaid wardley-beta block, and Mermaid draws every component label at the same default offset. On a real strategic map, where related components cluster, those labels land on top of each other and the map becomes correct and unreadable at the same time. ArcKit v5 fixes this with a PostToolUse hook that rewrites the label offsets automatically. This article steps through the algorithm behind it: a pure, deterministic placement engine that projects the map to pixels, generates 32 candidate slots per label, scores each against a weighted collision penalty, places the most-constrained labels first, runs a refinement pass, and converges to an idempotent result a Write hook can run on every save.

Technical 19 May 2026

The Leaf Node Problem: why your AI pilot is optimising the wrong thing

Most AI pilots in large organisations are funded by a team and accountable to a team. The team has KPIs. Six months later there is a write-up showing local improvement, the pilot is declared a success, and the enterprise outcome the company actually cares about does not move. Valentine et al. (2024) named the pattern; Gonzalez picked it up as Idea 10 of Organizing Intelligence at DeepMind. The architectural shorthand is the leaf node problem — AI tooling deployed along the org chart inherits the org chart's gravity, with three points of inertia (Team KPIs, the org chart itself, and siloed data schemas) pulling every pilot downward. Mapped on a Wardley Map, the lift is visible, and so is the cost.

Your Wardley Maps belong in Git: what Mermaid support changes

For most of the last decade, Wardley Maps lived on a whiteboard, in a PNG screenshot in a deck, or on a third-party site buried in a wiki. Mermaid 11.15.0 finally makes wardley-beta stable enough to use in production, so GitHub renders maps natively in any markdown file and Claude renders them in the artifacts side panel. That combination quietly resolves the biggest practical problem with Wardley Mapping: keeping the map alive after the meeting. Maps go through code review, get versioned alongside the system they describe, appear where the work happens, and read and write the same way AI assistants already do.
